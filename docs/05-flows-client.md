# 05 — Клиент: меню и флоу

## Меню клиента (uk)

- 📦 **Товари** — перегляд, пошук по артикулу/назві/категорії, сторінки.
- 🚚 **Створити ТТН** — полный FSM (ниже).
- 📬 **Відправлення** — список ТТН, статуси, скасування до відправки.
- 📊 **Статистика** — сьогодні / тиждень / місяць + залишок.
- 💬 **Звернення до менеджера** — онлайн-чат с дежурным (релей) + телефон.
- ⚙️ **Налаштування** — сповіщення (вкл/викл) + профіль.

ФОП/ключ НП клиент **не настраивает** — это делает персонал.

## Регистрация и подтверждение

- `/start` → запрос контакта (телефон) → ПІБ. Новый клиент — `status=pending`,
  `role=client`.
- Сразу после создания — **надёжное** уведомление владельцу(ам): из
  `OWNER_TELEGRAM_IDS` + всем `owner/manager` из БД; дедуп по telegram_id.
  Кнопки «Підтвердити» / «Відхилити» / «Заблокувати».
- Подтверждение: `status pending→active`, **edit_reply_markup** (кнопки исчезают)
  + edit_text с итогом, audit, push клиенту «активовано». Повторный тап — мягкое
  «вже оброблено».
- Клиент в `pending` видит «очікує підтвердження»; после активации — полное меню.

## 📦 Товари (только просмотр)

- Товары из БД/склада (ручное добавление недоступно).
- Карточка: **артикул, назва, категорія, кількість/залишок (с учётом резерва),
  ціна**.
- Поиск по артикулу / назві / категорії; пагинация + фильтр по категории.

## 🚚 Створити ТТН (FSM)

На каждом шаге — «◀ Назад / ✖ Скасувати». ТТН создаётся в НП только на финальном
шаге (NP-first), резерв — после успешного ответа НП.

1. **Старт.** Клиент жмёт 🚚 Створити ТТН.
2. **Вибір ФОП.** Несколько → выбор; один → авто; нет ФОП → стоп: «ФОП ще не
   налаштований, зверніться до менеджера».
3. **Вибір товарів.** Список своих товаров с `available > 0` (поиск/пагинация);
   позиции и количества в корзину; валидация `qty ≤ available`.
4. **Перевірка складу.** Показ корзины → «Підтвердити / Змінити».
5. **Розмір/вага посилки** — пресеты НП:
   - **Документи** — до 1 кг (32×23×2 см)
   - **Мала** — до 2 кг (до 120 см)
   - **Середня** — до 10 кг (до 120 см)
   - **Велика** — до 30 кг (до 120 см)
   - **Власні розміри** — до 1000 кг (до 300 см) → ввод своих габаритов/веса.
6. **Опис вантажу** — текст.
7. **Тип отримувача** — фізособа / юрособа (юр → назва ФОП/ТОВ + ЄДРПОУ при
   необходимости; в НП — PrivatePerson или Organization).
8. **Отримувач — телефон** (валидация укр. формата).
9. **Отримувач — ПІБ** (для юр — контактна особа).
10. **Місто** — поиск по НП API (кэш Redis), выбор из списка.
11. **Відділення** — список по городу, по номеру відділення, НП API.
12. **Платник за доставку** — клиент выбирает: отримувач (по умолч.) / відправник
    (ФОП). → NP `PayerType`.
13. **Спосіб оплати** — передоплата / накладений платіж (COD).
14. **Сума накладеного** — только если COD. Комиссию платит **отримувач**.
15. **Оцінкова (страхова) сума** — клиент вводит вручную (NP `Cost`); валидация
    > 0 и ≥ минимума НП.
16. **Фінальне підтвердження** — полная сводка → «Підтвердити / Змінити».
17. **Відправити:**
    - НП `InternetDocument.save` (отправитель = наш склад авто, контрагент = ФОП);
    - **успех:** `Shipment` + items, № ТТН/`np_ref`; резерв (`reserved += qty`,
      `ttn_reserve`); в этот момент заводится SLA-таймер (30 рабочих минут от
      создания, `sla_deadline` — см.
      [08-notifications-tracking-returns.md](08-notifications-tracking-returns.md));
      клиенту — подтверждение с № ТТН; **push менеджеру** (группа «Створені»);
    - **ошибка НП:** понятная ошибка, ничего не резервировать, повтор/изменение.
18. ТТН уходит в очередь менеджера и в клиентское 📬 Відправлення.

Маппинг полей на НП и онлайн-ценообразование — [09-novaposhta-api.md](09-novaposhta-api.md).

**Расхождение веса при отправке.** НП перевзешивает, **сама пересчитывает
стоимость и отправляет** — без отмены/беспокойства клиента. Платит платник. Бот
может тихо синхронизировать факт. вес/стоимость в карточку (без сообщений).

## 📬 Відправлення

- **Список** группами: **Створені / Підтверджені / Повернення**, сорт. по даті
  створення, пагинация. (Створені = клиент создал, ещё не собрано; Підтверджені =
  менеджер подтвердил получение и собирает/отправляет; Повернення = returning/
  returned.)
- **Поиск** по № ТТН / получателю / даті.
- **Карточка ТТН:** получатель, тип (фіз/юр), відділення, позиции, оплата/суммы,
  статус + таймстемпы трекинга, № ТТН.
- **Действия:** ❌ Скасувати (если статус < `dispatched` → возврат резерва,
  уведомления); 💬 Звернення до менеджера по этой ТТН.

## 📊 Статистика

- Без «всього відправлено». Периоды (Europe/Kyiv): **сьогодні / тиждень / місяць**.
- Исходы: **Відправлено** (≥ `dispatched`), **Повернення/Відмови**
  (`returning`/`returned`), **Втрати/Пошкодження** (`lost`/`damaged`).
- **Чисті продажі = Відправлено − Повернення − Втрати.**
- Залишок на складі + топ-артикулов.
- Выбор дня: быстрые кнопки последних дней + inline-календарь; разбивка по
  артикулам (отправлено и возвраты), итог чистых продаж за день.

## ⚙️ Налаштування

- **Сповіщення.** Тумблеры вкл/выкл: регистрация подтверждена; статусы ТТН/
  трекинг; остатки/низкий остаток.
- **Профіль.** Клиент может редактировать **ВСЕ свои данные**: телефон, ПІБ,
  назви ФОПов и реквизиты. Правка — через шаг подтверждения:
  1. «✏️ Змінити» → выбор поля → ввод (валидация).
  2. «було → стане» → «Підтвердити / Змінити».
  3. «Підтвердити» → сохранение + audit.
  (Ключ НП у ФОП — настраивает персонал.)
