# 04 — Склад и приёмка (Google Sheets)

Текущий прод-источник наличия — Google Sheets. Резерв и история
движений-отгрузок — Postgres. В Phase 7 чтение/списание вынесены за seam
`app/sheets/` (`INVENTORY_SOURCE=sheets|crm`), но фактическая рабочая реализация
пока именно Sheets. В боте раздел 📦 Склад даёт **просмотр остатков** и
**две кнопки-ссылки** (на книгу приймання и книгу складу).

## Модель: две книги, листы на клиента

### Книга «Склад»

- Общий склад, **лист на клиента** (напр. «Вася», «Антон», «Петя»).
- Источник правды: артикул, назва, категорія, кількість, ціна. Бот связан
  **только с этими листами**.
- **Только чтение** (Protected ranges): количество правит исключительно
  **Script/бот** (приёмка `+`, отгрузка `−`).
- **Исключение:** по запросу **владелец или dev** могут изменить количество
  (коррекция ошибки) — фиксируется в «Історія»/`audit_logs`.
- **Лист — на клиента (человека), не на ФОП.** У клиента может быть несколько
  ФОПов (SenderProfile), но остаток **общий** — один пул. При ТТН клиент выбирает
  ФОП, списание идёт из общего остатка.

### Книга «Приймання»

- **Лист на клиента** (та же разбивка), лист = **черновик ввода**.
- **Справочник под клиента:** категории/товары на листе приёмки клиента
  **тянутся из его листа «Склад»** (IMPORTRANGE → Data Validation/VLOOKUP) —
  у Васи видны только товары Васи, у Пети — Пети (кофе и колёса не пересекаются).
- **Сверка по накладной:** клиент при отправке товара авто-шлёт накладную
  менеджеру; менеджер физически сверяет (кол-во, брак). Брак/недостача в «Склад»
  не идут; клиенту — уведомление (претензия на НП).

## Синхронизация приёмка → склад: «Оброблено» + кнопка «Внести» + double confirmation

Реализация — `scripts/intake_apps_script.gs` (контейнер-bound к «Приймання»,
книга «Склад» по Script Property `STOCK_BOOK_ID`). Синхронизация — **только по
кнопке меню**, без авто-списаний по времени:

1. В строках черновика менеджер ставит галочку **«Оброблено»** на готовых позициях
   (брак не отмечает — или ставит Стан «брак», такие в склад не идут).
2. Меню **📦 Склад → «Внести оброблені → Склад»** → скрипт агрегирует по артикулу
   только отмеченные (годные) строки и показывает **сводку «что прибавится»**
   (**Підтвердити** = YES / **Змінити** = NO → правка и повтор).
3. **«Підтвердити»** → `Склад[артикул] += кількість` (новый артикул → новая
   строка) + запись в «Історія» → **очищаются именно перенесённые строки**;
   неотмеченные черновики и брак остаются.

Есть и пункт «Внести всі (годні) → Склад» (весь лист разом, минуя галочки).
Additive, прибавляется один раз; без перезаписи ячеек. Защита: если листа с таким
именем в «Склад» нет — внесение **отменяется с явной ошибкой** (имена листов
«Приймання»/«Склад» должны совпадать, см. `stock_sheet_key`), остаток не «теряется».

## UX приёмки (Google Sheets)

- Колонки: Дата · Артикул · Назва · Категорія · Кількість · Ціна · Накладна ·
  Стан (годне/брак) · Оброблено.
- **Живой справочник:** меню **📦 Склад → «🔄 Оновити довідник»** тянет
  Артикул/Назва/Категорія из листа «Склад» клиента в скрытую вкладку `_Довідник`
  и переустанавливает базовые дропдауны. Запускать после изменения ассортимента.
- **Зависимые дропдауны и автоподстановка** (`onEdit`):
  - выбор **Категорії** → список **Назв** в строке сужается до этой категории;
  - выбор **Назви** → авто-подставляются **Артикул** и **Категорія**;
  - выбор **Артикула** (дропдаун ищет по подстроке) → авто-подставляются
    **Назва** и **Категорія**.
- **Авто-дата:** при первой правке строки в пустую «Дата» ставится сегодняшняя
  (формат `ДД.ММ.РРРР`); правится вручную через точку, двойной клик — календарь.
- «Новий товар»: дропдауны нестрогие (`strict=false`) — можно ввести артикул/назву,
  которых ещё нет в «Склад» (создаст строку в «Склад» при внесении).
- Чекбоксы «Стан/брак» и «Оброблено», заморозка шапки.

## Минимизация человеческого фактора

- Защита главной книги (только Script/бот; правка кол-ва — owner/dev по
  исключению) — основной риск (порча источника правды) снят.
- Жёсткая валидация ввода (артикул/категория из выпадающих; новый — через явный
  блок; кол-во > 0; служебные колонки защищены).
- Подтверждение перед записью (сводка «что прибавится»), затем строки блокируются.
- Аудит и откат: каждая приёмка — в «Історія»; owner/dev могут откатить/исправить.

## Гибрид с Postgres

- Книга «Склад» хранит `quantity`; `reserved` (под открытые ТТН) и история
  движений-отгрузок — в Postgres (`stock_movements`).
- `available = quantity(Sheets) − reserved(Postgres)`.
- При «відправлено» бот пишет списание в лист «Склад» (Sheets API); приёмка
  прибавляет (Apps Script) — над одним листом; нагрузка лёгкая (≤20 клиентов).

## Адресное хранение

Наш единственный склад = точка отправления; авто-подвязка отправителя при ТТН;
ячеек/локаций нет.

## Раздел 📦 Склад в боте (менеджер/владелец)

- **Сценарий 1. Перегляд залишків.** Бот читает «Склад» клиента → список товаров
  (артикул/назва/категорія/кількість/ціна), `available`/`reserved`, поиск,
  пагинация.
- **Сценарий 2. Две ссылки.** 📥 Таблиця приймання (ввод данных) и 📦 Таблиця
  складу (только просмотр). Синхронизация — по кнопке «Внести» (выше).
- **Сценарий 3. Рух залишків.** История движений бота (`ttn_dispatch`,
  `ttn_return`, `manual`) по товару/клиенту: тип, было→стало, кто, когда; фильтры.
  (Приёмки — в листе «Історія».)
