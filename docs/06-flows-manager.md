# 06 — Менеджер: меню и флоу

## Меню менеджера (uk)

- 🟢 **Я на зв'язку** — открытие смены: нажатие = авторизация менеджера на день +
  включение дежурства (`/start` лишь показывает меню, смену открывает только эта
  кнопка); выключение автоматическое при закрытии отделения (см.
  [10-support-duty.md](10-support-duty.md)).
- 📬 **Відправлення** — очередь ТТН (ниже).
- 📦 **Склад** — перегляд остатков + 2 ссылки (приймання/склад), рух залишків
  (см. [04-warehouse-sheets.md](04-warehouse-sheets.md)).
- 👥 **Клієнти** — список/карточка, підтвердити/заблокувати/видалити, редагувати
  (по флагу), ФОП/ключ НП.
- 💬 **Підтримка** — треды дежурства (свои/назначенные), ответы клиентам.
- 📊 **Звіти** — сводка по отправкам, по клиенту, склад (см.
  [08](08-notifications-tracking-returns.md) и ниже).

Принцип: **списание резерва только авто по трекингу НП** (ручной отметки
«відправлено» нет).

## 📬 Відправлення

**Реальный процесс:**
1. Клиент формирует ТТН → НП `InternetDocument.save` → **push менеджеру**.
   Статус `created` (группа «Створені»).
2. Менеджер видит новую ТТН в «Створені».
3. **«Підтвердити»** = подтверждает, что **получил ТТН**, и **начинает собирать**
   посылку → `confirmed` (группа «Підтверджені»). (Створені = ещё НЕ собранные.)
4. **Вне бота:** относит посылку в отделение НП и сдаёт оператору НП.
5. Статус в боте обновляется **авто по API-ключу (трекинг)**.

**Сценарий 1. Очередь ТТН.** Группы **Створені / Підтверджені / Повернення**,
сорт. по даті створення, счётчики, пагинация. Фильтры: статус/клиент/дата; поиск
(№ ТТН / получатель). По каждой ТТН — дедлайн SLA и индикатор «встигаємо /
прострочено» (см. [08-notifications-tracking-returns.md](08-notifications-tracking-returns.md)).

**Сценарий 2. Карточка ТТН.** Клиент, ФОП-отправитель, получатель (ПІБ/тел),
місто/відділення, позиции (артикул×кол-во), вес, оценка, оплата, статус +
таймстемпы, дедлайн SLA + индикатор «встигаємо/прострочено», № ТТН. Действия: ✅ **Підтвердити** (`created → confirmed`, на склад
не влияет), ❌ Скасувати (если статус < `dispatched`), 🔄 **Повернення замовлення**
(если `returning`/`returned`), 💬 Клієнт.

**Сценарий 3. Отправка (авто, трекинг).** Воркер опрашивает трекинг; при переходе
через `dispatched`: списание резерва (`quantity −= qty`, `reserved −= qty`,
движение `ttn_dispatch`), статус → `dispatched` + `dispatched_at`, push клиенту.
Далее `in_transit`/`arrived`/`delivered`/`returned` — трекингом.

**Сценарий 4. Отмена ТТН (до отправки).** При статусе < `dispatched`:
подтверждение → НП `InternetDocument.delete` → возврат резерва (`reserved −= qty`,
`ttn_cancel`) → статус `cancelled` → push клиенту → audit. Если уже `dispatched` —
отмена недоступна.

**Сценарий 5. Связь с клиентом по ТТН.** 💬 Клієнт → тред поддержки, привязанный
к этой ТТН.

**Сценарий 6. Фильтры/поиск** — перестраивают список.

## 👥 Клієнти (менеджер и владелец)

**Сценарий 1. Список.** Группы/фильтр по статусу (Очікують / Активні /
Заблоковані / Видалені), поиск (ПІБ/телефон), пагинация, счётчики.

**Сценарий 2. Подтверждение нового клиента.** Из push **или** из «Очікують»:
**«Підтвердити»** → `status active`, кнопки исчезают (`edit_reply_markup`), push
клиенту «активовано», audit. **«Відхилити»** → отклонение + уведомление.

**Сценарий 3. Карточка клиента.** Профиль: ПІБ, **телефон**, статус, дата
регистрации. Разделы: **його ТТН**, **залишки/
товари** (его лист «Склад» + ссылка), **ФОПи**. Действия: ✏️ Редагувати ·
Заблокувати/Розблокувати · Видалити · ФОП.

**Сценарий 3a. Редактирование карточки.** Исправление данных (ПІБ, телефон…).
Доступно менеджеру (по флагу `can_edit_clients`), владельцу и dev; правка в audit.

**Сценарий 4. Блокировка/разблокировка.** «Заблокувати» → `status blocked`
(доступ закрыт); «Розблокувати» → `active`; audit. **Клиента не уведомляем.**

**Сценарий 5. Удаление (мягкое).** Пометка archived/deleted, данные сохраняются,
доступ закрыт, обратимо, audit. **Клиента не уведомляем.**

**Сценарий 6. ФОП / ключ НП (настраивает персонал).** Добавить/редактировать ФОП:
поля SenderProfile (см. [09-novaposhta-api.md](09-novaposhta-api.md)). Проверка
ключа через НП API при сохранении; несколько ФОП на клиента.

## 📊 Звіти (менеджер)

Те же исходы, что в статистике (Відправлено / Повернення-Відмови /
Втрати-Пошкодження). Период: сьогодні/тиждень/місяць/произвольный (Europe/Kyiv).
Данные: shipments — Postgres, остатки — Sheets.

- **Сценарий 1. Сводка по отправкам.** За период — итоги по исходам (всего и **по
  каждому клиенту**), «чисті продажі» = відправлено − повернення − втрати.
- **Сценарий 2. По клиенту.** Метрики/исходы, топ-артикулы, текущий остаток.
- **Сценарий 3. Склад.** Сводка по единицам остатков: позиции/единицы по клиентам
  и общее количество.

Аналитика владельца (динамика, по менеджерам, поддержка, финансовый отчёт по
fee-формуле, опоздавшие ТТН по SLA) — [07-flows-owner-dev.md](07-flows-owner-dev.md).
