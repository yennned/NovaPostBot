# 01 — Обзор, контекст и ключевые решения

## Контекст

Telegram-бот личного кабинета фулфилмента Новой Почты. Работа ведётся **только**
в `/Users/yenin/Desktop/NovaPostBot`.

**Цель:** рабочий, поддерживаемый бот с ролевой моделью (клиент → менеджер →
владелец), god-mode для разработки, корректными уведомлениями/подтверждением
клиентов и аккуратным процессом разработки в две руки (я + Степан) через GitHub
с защищённым `main`.

## Ключевые решения

| Тема | Решение |
|------|---------|
| Рабочая папка | Только `/Users/yenin/Desktop/NovaPostBot`, с нуля |
| Стек | Python 3.12 · aiogram 3 · **PostgreSQL** (SQLAlchemy async + Alembic) — вся БД · **Google Sheets** — учёт склада · Redis (FSM/кэш) · Docker |
| Хранилище | **Postgres** — вся база (клиенты, ФОП, ТТН, поддержка, уведомления, аудит/логи). **Google Sheets** — **только учёт склада** (остатки «Склад» лист на клиента + приёмка). CRM/WMS для склада — позже |
| Интерфейс | MVP — классические inline/reply-кнопки; Mini App (WebApp) заложен как эскейп для формы ТТН/каталога/аналитики; бэкенд API-first |
| Язык бота | Только украинский (uk). Тонкий слой строк, без переключателя языка |
| Роли | **client → manager → owner**, строго сверху вниз. (`operator` — возможно в будущем, отдельной миграцией) |
| Менеджер | Вся операционка: приёмка, обработка/отправка ТТН, поддержка (дежурство), управление клиентами, отчёты. Push по каждой новой ТТН |
| Dev-доступ | Полный god-mode по allowlist Telegram ID из `.env` (`/as <role>`, impersonation, kill-switch) |
| Остатки | Источник правды — Google-книга «Склад» (лист на клиента). Бот связан только с ней |
| Приёмка | Вне FSM бота; в книге «Приймання» (лист на клиента) → приплюсовывается в «Склад» кнопкой с двойным подтверждением |
| Бот↔склад | Бот читает «Склад», отражает остаток; при отгрузке системно уменьшает |
| Авторизация | Только телефон (`request_contact`) для всех ролей |
| ТТН | Ключ НП клиента; несколько ФОП/отправителей (SenderProfile); ФОП и ключ настраивает персонал; бот — единственный канал отправок |
| Товары клиента | Клиент НЕ добавляет товар вручную — тянется из БД/склада |
| Статистика | Сегодня/неделя/месяц; исходы Відправлено/Повернення/Втрати |
| SLA | Відправка за 30 хв от создания ТТН (счёт только в рабочее время; вне часов — старт со след. рабочего дня); «встиг» = `dispatched` по трекингу до дедлайна; промах → наша комиссия за заказ = 0 |
| Финотчёт владельца | `fee(ТТН) = 20 грн + (кількість одиниць − 1) × 1 грн`; отчёт за день/неделю/месяц: #ТТН, #единиц, сумма fee (минус бесплатные по SLA); отдельно — количество опоздавших ТТН (SLA >30 хв) + список их номеров (они же бесплатные) |
| PII | Персонал видит телефон/ФІО **открыто** |
| Git | GitHub, ветка на задачу, в `main` только через PR, защищённый main |
| Прогресс | `PROGRESS.md` — после каждого локального коммита |

## Объёмы и сайзинг

Входные: ≤ **20 клиентов** на старте, ≤ **15 000 ТТН/мес**.

- **Отправки:** ~500 ТТН/день, ~1/мин (пики 2–3/мин) — нагрузка очень лёгкая.
- **Позиции ТТН:** ~30–45 тыс. строк/мес.
- **Наименования на складе:** ~2 000–20 000 (планово ~10 000 SKU на всех).
- **Google Sheets (склад):** книга «Склад» ~120 тыс. ячеек из лимита 10 млн
  (~1%); API-квоты с запасом. Узкого места нет.
- **PostgreSQL:** < 1 млн строк/год (shipments+items+audit+поддержка) — мелочь,
  без партиционирования.
- **Compute:** бот+воркер+Redis на 1–2 vCPU / 2–4 ГБ RAM с запасом.

Узкое место — скорость API НП, а не БД. Поэтому лёгкая очередь (Redis+worker) и
кэш справочников НП.

## Хостинг (вариант A, цены на 14.06.2026)

- **VPS:** Hetzner **CPX21** (3 vCPU / 4 ГБ / 80 ГБ); держит бот + воркер + Redis
  в Docker. (Экономнее — CPX11.) ⚠️ Hetzner поднял цены до 37% с 01.04.2026
  (CPX22 теперь €7.99/мес) — точную цену CPX21 после повышения сверить при заказе.
- **БД:** managed Postgres **Neon** — на проде **план Launch с отключённым
  scale-to-zero** (БД всегда «тёплая»). Launch — pay-as-you-go **без минимума $5**
  ($0.106/CU-час, $0.35/ГБ-мес, PITR до 7 дней); при наших объёмах (~0.25 CU
  baseline) — **низкие единицы $/мес**. Автобэкапы/PITR/SSL на стороне Neon.
  **Neon Free — НЕ для прода:** обязательный scale-to-zero через 5 мин простоя
  (не отключается) даёт холодные старты ~0.5–неск. сек на каждый запрос после
  простоя; Free годится только для dev/staging.
- **Redis:** на VPS в Docker = $0. **Telegram/НП/Sheets API — бесплатно.**
- **Итого инфраструктура: ~€10–15/мес** (Hetzner ~€8 + Neon Launch низкие
  единицы $).
- Код провайдеро-независим (`DATABASE_URL`/`REDIS_URL`) — смена провайдера без
  правок. Managed-БД с Фазы 0.

**Альтернативы (отклонены):** DigitalOcean (одно окно, ~$27–48/мес — дороже);
один VPS со self-hosted Postgres (~$5–8/мес, но ручные бэкапы — риск для прода).
