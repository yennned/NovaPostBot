# AGENTS.md — контекст для AI-агентов (Codex code review и др.)

Проект — Telegram-бот личного кабинета фулфилмента Новой Почты.
Полная карта — в [CLAUDE.md](CLAUDE.md), детальный план — в [`docs/`](docs/).

**Стек:** Python 3.12 · aiogram 3 · PostgreSQL (SQLAlchemy async + Alembic) ·
Redis · Google Sheets API (service-account) · Nova Poshta API · Docker.

**Хранилище (гибрид):** PostgreSQL — вся БД (users, sender_profiles, shipments,
stock_movements, support, audit_logs). Google Sheets — только склад
(«Склад» read-only + «Приёмка» черновик). `available = Склад(Sheets) − reserved(PG)`.

## Review guidelines

Ревьюй **только дифф PR**. Пиши по-украински, кратко и конкретно, со ссылками на
файлы/строки. Если серьёзных проблем нет — коротко подтверди. На что смотреть:

- **Границы слоёв.** `app/services` и `app/db` **не импортируют `aiogram`**
  (API-first слой; это же проверяет CI). Бизнес-логика — в services, не в handlers.
- **Секреты и шифрование.** Ключ НП хранится только зашифрованным (Fernet).
  Ключи/токены/телефоны не должны попадать в логи, тексты ошибок, коммиты, тесты.
  Никаких секретов в репозитории — только через env / GitHub Secrets.
- **RBAC.** Иерархия строго `client → manager → owner` + dev god-mode по allowlist
  `DEV_TELEGRAM_IDS`. Права менеджера — per-flag в `users.permissions`. Проверяй,
  что хендлеры/сервисы не дают действие мимо проверки роли/флага.
- **Остатки.** Списание остатка — **только авто по трекингу НП**, руками нигде не
  списываем. `available` считается как склад минус reserved, не путать источники.
- **Часовой пояс** — Europe/Kyiv; даты/дедлайны/SLA не должны считаться в UTC/локали.
- **Миграции.** Alembic построчно не обязателен, но проверь обратимость
  (upgrade→downgrade) и что автоген не тянет лишних DROP/ALTER.
- **Язык.** Тексты бота — украинский; комментарии/докстринги — русский.
- **Гонки/идемпотентность.** Воркер (поллинг НП, SLA, уведомления) и синк склада
  должны быть устойчивы к повторам и параллельным апдейтам.

## Процесс

Ветка на задачу → PR в `main` (защищённый, зелёный CI обязателен). Коммиты
точечные (без `git add .`), conventional. После каждого коммита — запись в
[PROGRESS.md](PROGRESS.md). Тесты идут на **реальном Postgres** (не sqlite).
